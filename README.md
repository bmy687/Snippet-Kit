# Snippet Kit

**一个为"需要在别人面前敲代码的人"而生的 VS Code 插件。**

提前把代码粘贴保存，现场按键盘任意键，预设代码就会逐字符出现——看起来完全就是你在实时编写。

不用背代码。不用复制粘贴。不会敲错。

---

## 演示效果

```
1. 保存一段代码 → 命名为"登录模块"
2. 绑定到快捷键槽位 1
3. 现场按 Ctrl+Alt+1 启动
4. 随便按键盘，代码逐字符出现
5. 按 Ctrl+Alt+0 停止，恢复正常输入
```

观众看到的：你在流畅地写代码。

实际发生的：你按什么键都无所谓，输出的是你提前准备好的内容。

---

## 功能特性

- **粘贴即保存** — 粘贴代码，取个名字，点 Save。没有任何多余的配置。
- **快捷键槽位** — 最多 5 个片段绑定到 `Ctrl+Alt+1` ~ `Ctrl+Alt+5`，演示中随时切换，无需打开面板。
- **即时停止** — `Ctrl+Alt+0` 立刻恢复正常输入，随时可以切回真实打字。
- **隐蔽设计** — 启动后侧边栏自动关闭，状态栏只显示行号，看起来就像编辑器自带的指示器。插件名、命令名全部使用中性词汇，扩展列表里看到的只是一个普通的代码片段管理工具。
- **语言无关** — 不限制编程语言和文件类型。JavaScript、Python、Java、Go、Rust、SQL，什么都行。
- **自然节奏** — 每次按键随机输出 1~3 个字符，模拟真实打字的速度波动，不会匀速得像机器人。

## 安装

### 方式一：下载 VSIX 安装

1. 直接下载.vsix文件
2. 打开 VS Code，按 `Ctrl+Shift+P`
3. 输入 `Extensions: Install from VSIX...`
4. 选择下载的文件，安装完成

## 使用方法

### 1. 保存片段

打开左侧活动栏的 **Snippets** 面板：

- 在 Name 输入框填写片段名称
- 在下拉框选择快捷键槽位（可选，No key 表示不绑定）
- 在文本框粘贴你要演示的代码
- 点击 **Save**

可以保存任意数量的片段，每个槽位只能绑定一个。

### 2. 启动演示

三种方式，任选其一：

| 方式 | 操作 |
|------|------|
| 快捷键 | `Ctrl+Alt+1` ~ `Ctrl+Alt+5` 直接启动对应槽位 |
| 面板按钮 | 点击片段旁边的 **Use** 按钮 |
| 命令面板 | `Ctrl+Shift+P` → `Snippet Kit: Apply Snippet` → 选择片段 |

启动后侧边栏自动关闭，状态栏出现一个低调的行号指示器。

### 3. 开始"敲代码"

随便按键盘上任何键，预设代码会逐字符输出。按什么键都无所谓。

### 4. 停止

三种方式：

- 代码输出完毕后自动停止
- 按 `Ctrl+Alt+0` 手动停止
- 点击状态栏的行号指示器

停止后立刻恢复正常输入，无需重启或禁用插件。

## 快捷键

| 快捷键 | 功能 |
|--------|------|
| `Ctrl+Alt+1` ~ `5` | 启动对应槽位的片段 |
| `Ctrl+Alt+0` | 停止演示，恢复正常输入 |
| `Ctrl+Alt+H` | 切换侧边栏显示 / 隐藏 |

Mac 用户将 `Ctrl` 替换为 `Cmd`。

## 适用场景

**技术大会 / Meetup 演讲** — 做 live coding 演示时把注意力放在讲解上，不用担心敲错。

**编程教学和课程录制** — 学生看到的是干净流畅的编码过程，省去反复剪辑的时间。

**项目汇报和技术评审** — 向领导、客户或甲方现场展示技术实现，从容不迫。

**技术面试** — 给候选人演示题目思路或示例代码时保持流畅。

**技术自媒体** — 在 B 站、YouTube 等平台录制编程教程，一遍过就是成品。

**售前演示和路演** — 现场写代码比 PPT 更有说服力，前提是不能卡壳。

## 与其他方案的对比

| 方案 | 问题 |
|------|------|
| 背代码 | 能背 10 行，背不了 100 行，认知负担影响讲解质量 |
| 复制粘贴 | 一眼就能看出来，代码突然出现一大片 |
| Hacker Typer | 需要录制操作过程，录错要重来，只支持单文件 |
| 扩展屏抄代码 | 打字速度和眼神会暴露你在看别的地方 |
| **Snippet Kit** | 粘贴保存即可，多片段快捷键切换，即时停止恢复 |

## 已知限制

- 演示模式下 `type` 命令被覆盖，无法输入中文。建议将中文注释部分拆出来，停止演示后手动输入。
- 每个快捷键槽位只能绑定一个片段，最多支持 5 个槽位。

---

## 技术实现

核心原理是通过 `vscode.commands.registerCommand('type', ...)` 覆盖 VS Code 的默认输入命令。当演示模式激活时，用户的每一次按键不再执行默认的字符插入，而是从预设代码中取出下一段字符插入到编辑器中。停止演示时释放命令覆盖，恢复默认行为。
